# 무료운세 데이터셋 생성/검수 프롬프트

이 문서는 `app/src/main/assets/life_records_*.jsonl` 품질을 높이기 위한 작업용 프롬프트입니다.
한 번에 1,000개를 만들기보다 20개 단위로 생성하고, 별도 검수 프롬프트를 한 번 더 돌린 뒤 JSONL에 반영합니다.

## 생성 단위

- 권장 단위: 20개
- 출력 형식: JSON 배열만
- 이후 반영 시 각 객체를 한 줄 JSON으로 변환해 `life_records_0000_0999.jsonl` 같은 범위 파일에 넣습니다.

## 생성 프롬프트

너는 운세·성향 분석 데이터셋을 만드는 전문 카피라이터이자 JSON 콘텐츠 설계자다.

내가 제공하는 숫자별 성향 데이터와 조합값을 바탕으로, 각 케이스별 운세 해석 JSON을 작성해라.

작성 목표는 다음과 같다.

1. 문장은 자연스럽고 사람이 읽기 쉬워야 한다.
2. 각 케이스는 비슷해 보이더라도 반드시 고유한 흐름을 가져야 한다.
3. 같은 숫자가 반복되더라도 초년, 중년, 말년에서 의미가 다르게 드러나야 한다.
4. destiny는 전체 인생의 기본 기질로 해석한다.
5. early는 성향이 형성되는 시기, middle은 책임과 현실 속에서 드러나는 시기, late는 정리·관계·삶의 태도로 남는 시기로 해석한다.
6. 단순히 키워드를 이어 붙이지 말고, 각 숫자가 서로 만났을 때 생기는 긴장감과 보완점을 추론해서 작성한다.
7. 각 항목은 긍정적 해석, 주의 흐름, 실천 조언 순서로 자연스럽게 이어져야 한다.
8. 지나치게 단정하거나 공포를 주는 표현은 피하고, 부드러운 상담형 문체로 작성한다.
9. "~할 수 있습니다"만 반복하지 말고 문장 구조를 다양하게 사용한다.
10. 이전 케이스와 동일한 문장을 재사용하지 않는다.

각 케이스마다 다음 3가지를 먼저 내부적으로 정한 뒤 글을 써라.

1. 이 조합의 중심 성향
2. 이 조합의 중심 갈등
3. 이 조합의 성장 방향

이 3가지는 JSON에 직접 출력하지 말고 `earlyText`, `middleText`, `lateText`, `lifeText`, `summaryText`, `oneLineAdvice` 안에 자연스럽게 녹여라.

각 케이스마다 반드시 하나의 중심 갈등과 하나의 성장 방향을 다르게 설정해라.
중심 갈등은 `cautionKeywords`에서 뽑되, 모든 `cautionKeywords`를 억지로 넣지 말고 해당 조합에 가장 어울리는 1~2개만 선택해라.
성장 방향은 `coreKeywords`에서 뽑되, 해당 케이스만의 행동 조언으로 구체화해라.

작성 기준:

- `lifeTitle`: 기본 기질, 초년 흐름, 중년 흐름, 말년 흐름이 자연스럽게 드러나는 제목. 기계적으로 반복하지 않는다.
- `earlyText`: 성향이 만들어지는 방식, 습관, 감정 반응, 배움의 방식. 4~5문장.
- `middleText`: 일, 책임, 관계, 선택, 성과. 초년과 같은 말을 반복하지 말고 현실 문제로 확장. 4~5문장.
- `lateText`: 관계 정리, 삶의 태도, 내려놓음, 지혜, 생활 리듬. 4~5문장.
- `lifeText`: 전체 흐름을 하나의 서사로 연결. 숫자 나열 금지. 4~5문장.
- `summaryText`: 전체 흐름 1문장, 핵심 조언 1문장. 총 2문장.
- `oneLineAdvice`: 해당 조합에만 맞는 실천 조언. 1~2문장.

중복 방지 규칙:

- 같은 문장을 다른 케이스에 그대로 반복하지 않는다.
- 같은 destiny가 반복되어도 문장을 복사하지 않는다.
- 같은 early, middle, late 숫자가 등장해도 시기와 조합에 따라 다르게 해석한다.
- 모든 케이스에 공통으로 들어가는 안전한 문장을 반복하지 않는다.
- 가능성+개척+조화 조합은 "방향을 정한 뒤 관계를 부드럽게 조율하는 흐름"처럼 쓰고, 가능성+개척+집중 조합은 "넓은 가능성 중 하나를 선택해 깊게 파고드는 흐름"처럼 다르게 쓴다.

문체 규칙:

- 부드럽고 이해하기 쉬운 한국어.
- 너무 신비주의적이지 않게.
- 단정하지 않고 참고용 해석처럼 작성.
- 일, 관계, 선택, 생활 습관 같은 구체적인 표현을 섞는다.
- "운명적으로 반드시", "무조건", "실패한다" 같은 단정적 표현은 피한다.
- 한 문장을 너무 길게 쓰지 않는다.

출력 규칙:

- JSON 배열만 출력한다.
- 마크다운 설명, 주석, 코드블록은 넣지 않는다.
- key 이름은 입력 구조와 동일하게 유지한다.
- 각 text는 4~5문장.
- `summaryText`는 2문장.
- `oneLineAdvice`는 1~2문장.

입력 데이터:

`[여기에 20개 입력 데이터 붙여넣기]`

## 검수 프롬프트

아래 JSON 데이터셋을 검수하고 수정해라.

검수 기준:

1. 서로 다른 케이스에 동일하거나 거의 같은 문장이 반복되는지 확인한다.
2. `earlyText`, `middleText`, `lateText`의 관점이 구분되는지 확인한다.
3. `destiny`, `early`, `middle`, `late`의 조합이 실제로 반영되었는지 확인한다.
4. `oneLineAdvice`가 너무 일반적이거나 다른 케이스에 붙여도 어색하지 않은지 확인한다.
5. 문장이 너무 추상적이면 구체적인 생활 장면을 넣어 수정한다.
6. 한국어 띄어쓰기와 문장 흐름을 자연스럽게 다듬는다.
7. JSON 형식이 깨지지 않게 유지한다.

수정 방식:

- 기존 구조와 key 이름은 절대 바꾸지 않는다.
- 의미가 겹치는 문장은 새롭게 다시 쓴다.
- 과하게 불안감을 주는 문장은 부드러운 상담형 문장으로 바꾼다.
- 결과는 수정된 JSON만 출력한다.

검수할 JSON:

`[여기에 생성 결과 붙여넣기]`

## 현재 앱에서 필요한 life record key

`life_records_*.jsonl`의 각 줄은 아래 key를 유지해야 합니다.

```json
{
  "code": "",
  "destiny": 0,
  "early": 0,
  "middle": 0,
  "late": 0,
  "destinyProfileKey": 0,
  "lifeTitle": "",
  "earlyText": "",
  "middleText": "",
  "lateText": "",
  "lifeText": "",
  "summaryText": "",
  "keywords": [],
  "cautionKeywords": [],
  "oneLineAdvice": ""
}
```

## 현재 앱에서 필요한 destiny profile key

`destiny_profiles.json`과 `destiny_profiles_gendered.json`은 아래 key를 기본으로 사용합니다.

```json
{
  "destiny": 0,
  "title": "",
  "polarity": "",
  "coreKeywords": [],
  "cautionKeywords": [],
  "destinyText": "",
  "oneLineAdvice": ""
}
```

현재 앱은 확장 구조도 허용합니다. `destinyText`가 없으면 아래 필드를 순서대로 합쳐 성향 본문으로 사용합니다.

```json
{
  "summary": "",
  "strength": "",
  "caution": "",
  "actionGuide": ""
}
```
